Component ecosystem

组件矩阵围绕同一个控制面协作

Control Plane 提供治理模型和多协议入口，Console、Controller、Sidecar、SDK 与 Specification 分别覆盖可视化管理、Kubernetes 接入、数据面执行、Proxyless 接入和开放标准。

阅读组件文档查看 GitHub organization

Control Plane

核心控制面，负责服务发现、配置中心、治理规则、命名空间和多协议入口。

  • HTTP / gRPC / xDS / Nacos / Apollo / Eureka
  • 规则版本化、灰度下发、细粒度鉴权

Console

面向运维与平台团队的控制台，管理服务、命名空间、配置、用户与权限策略。

  • TDesign React + Vite
  • 服务治理资源的可视化入口

Rust SDK

Proxyless Service Governance 的 Rust 接入方式，覆盖注册发现、配置中心与治理验证。

  • crate: pole_rust = "0.2.0"
  • Rust >= 1.63.0

Kubernetes Controller

连接 Kubernetes 与 Lattice Hub，支持 Service 同步与 Sidecar 自动注入。

  • 全量 / 按需同步
  • dns / mesh 两种 sidecar 模式

Pingora Sidecar

基于 Pingora 的轻量数据面，提供 HTTP、HTTP/2、gRPC-h2c 转发与治理扩展点。

  • 前缀路由与轮询负载均衡
  • ACL、限流、熔断、指标扩展骨架

Specification

服务治理标准与 protobuf 协议定义，覆盖服务管理、流量治理、容错、安全与 AI Native。

  • Java / Go / Rust 生成入口
  • MCP server proto
