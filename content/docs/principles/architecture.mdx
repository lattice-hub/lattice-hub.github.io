---
title: 四层控制面架构
description: API 服务层、业务逻辑层、缓存层和存储层如何协同支撑服务治理。
---

Lattice Hub 控制面按四层组织：API 服务层、业务逻辑层、缓存层、存储层。

## API 服务层

API 服务层对外承载 HTTP、gRPC、xDS、Nacos、Apollo、Eureka 等协议入口。它的职责是协议适配、请求解析、基础鉴权和向业务逻辑层转发。

## 业务逻辑层

业务逻辑层组织服务发现、配置中心、治理规则、命名空间和 AI Native Registry 等领域能力。它决定资源的校验、状态变更、发布和灰度流程。

## 缓存层

缓存层为查询和订阅提供低延迟数据视图。现有资料记录了 19 类缓存、1 秒增量刷新和 `lastMtime - 5s` 查询窗口保护。

## 存储层

存储层负责持久化服务治理资源、配置、规则和权限数据。现有实现包含 MySQL 与 bbolt 等存储相关依赖。
