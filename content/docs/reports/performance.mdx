本报告首版只记录已有事实和待实测计划。当前没有可引用的正式压测结果，因此不写吞吐、延迟、资源占用等推测数字。

## 已确认配置事实

| 范围 | 当前事实 |
| --- | --- |
| 缓存刷新 | 19 类缓存，后台每 1 秒增量刷新 |
| 时间窗口保护 | 增量查询使用 `lastMtime - 5s` |
| 配置中心分页 | 最大分页 100 |
| 配置内容 | 最大 20,000 字符 |
| 批处理示例 | 队列 10000，等待 32ms，最大批量 128，并发 128 |
| MySQL 连接池示例 | `maxOpenConns=100`，`maxIdleConns=10` |
| 健康检查示例 | `slotNum=30`，最小 10s，最大 60s |

## 待实测指标

- Control Plane 服务发现查询吞吐、P95/P99 延迟和缓存命中表现。
- 配置发布、灰度和回滚链路延迟。
- SSE/长轮询监听在大量客户端下的连接数、内存和 CPU 使用。
- Sidecar 在 HTTP/1.1、HTTP/2 和 gRPC-h2c 转发下的吞吐与尾延迟。
- Kubernetes Controller 全量同步与按需同步的事件处理耗时。

## 报告原则

性能页必须区分“已确认配置事实”和“压测结论”。没有可复现实验命令、环境说明和原始数据前，不把任何数字写成性能结论。
