title: 灰度发布
description: 配置和治理规则在 Lattice Hub 中进行灰度、回滚和版本管理的实践入口。
---

灰度发布的核心是把“编辑态”和“生效态”分开。规则或配置可以先在控制面中完成编辑和校验，再通过发布版本进入客户端或数据面可见的 active 视图。

## 实践要点

- 对配置和治理规则都保留版本。
- 发布前完成权限检查、资源存在性校验和参数校验。
- active release 是客户端读路径的核心，不直接让客户端读取管理态规则。
- 回滚、停止灰度和删除发布版本都要进入审计和历史记录。
