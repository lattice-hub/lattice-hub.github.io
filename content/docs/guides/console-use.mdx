---
title: 控制台使用
description: Console 在服务治理、配置、规则发布和权限管理中的使用边界。
---

Console 是平台团队操作 Lattice Hub 的主入口。它不应被理解成独立系统，而是控制面资源的可视化管理层。

## 主要任务

| 任务 | 说明 |
| --- | --- |
| 服务管理 | 查看命名空间、服务、实例和运行状态。 |
| 配置管理 | 创建配置、发布版本、灰度、回滚并监听变更。 |
| 治理规则 | 管理路由、限流、熔断、泳道、镜像、Mock 和鉴权规则。 |
| 权限管理 | 维护用户、角色、策略和资源访问边界。 |
| 操作追踪 | 通过历史记录和观测链路追踪配置与规则变化。 |

## 使用原则

- 管理态编辑和运行态发布分开处理。
- 治理规则发布前先确认资源存在性和权限点。
- 对配置和治理规则保留灰度、回滚和历史记录。
- 平台级操作优先通过 Console 完成，业务运行时通过 SDK、Controller 或 Sidecar 消费结果。
